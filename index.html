Functions

The Building Blocks of Code

Matt Gray

What is a function?

Functions break down problems into manageable parts.

Generally, the craft of programming is the factoring of a set of requirements into a set of functions and data structures.

Functions reduce complexity.

Complexity is the programmer's enemy. Functions help hide complexity by creating an abstraction around an operation and giving it a name.

Coloring errors red

For example, suppose you want to color page elements blood-red on error, using jQuery:


$( 'input[name="first_name"]' ).css( { color: '#d00' } );

$( '#message' ).css( { color: '#d00' } );

// Show and make em bleed
$( '#message ul li' ).show( ).css( { color: '#d00' } );

Using a function

Instead, use a self-documenting function:


function color_for_error ( elements ) {
	return $( elements ).css( { color: '#d00' } );
}

We no longer need to think about what's under the hood.


color_for_error( 'input[name="first_name"]' );

color_for_error( '#message' );

color_for_error( '#message ul li' ).show( );

If you can remember the function name, you are done.

It is also easy to guess what the function does by its name.

Functions reduce duplicate code.

Don't repeat yourself if you don't have to. If you find that you are writing the same code in many places, consider using a function instead.

Back to our example

Now the customer wants the color to fade in on error.

Without the function, you need to change many lines like:


// requires jQuery UI
$( '#message ul li' ).animate( { color: '#d00' }, 500 );

With the function, you can make the change in one place:


function color_for_error ( elements ) {
	return $( elements ).animate( { color: '#d00', 500 } );
}

Now all error coloration will animate.

Functions improve reliability and maintainability.

Think about the previous example:


function color_for_error ( elements ) {
	return $( elements ).animate( { color: '#d00', 500 } );
}
  • It would be easy to change the color or animation
  • Using the function enforces consistency
  • Reading the code gives clues about the what and why

Parts of a function


function add ( argument1, argument2 ) {
	var sum  =  argument1 + argument2;
	return sum;
}
  1. function
    The reserved word that begins a function definition
  2. add
    Optional function name
  3. argument1 and argument2
    Optional arguments (or parameters); the input
  4. The function body is between the curly braces, { }
  5. return
    Optional; ends execution, may include a return value, the output

Functions are objects.

You can create functions, assign them to variables, store them in arrays, and use them like any other value.


// This function has no name, but is referenced by add_one
var add_one  =  function ( a, b ) {
	return a + b;
};

// my_add_one and add_one now reference the same function
var my_add_one  =  add_one;

// An array of functions!
var my_functions  =  [ add_one, my_add_one ];

my_functions[0]( 1, 2 ); // === 3

Invoking a function

There are four ways to invoke a function:

  1. Function

var add_one  =  function ( a, b ) {
	return a + b;
};

add_one( 1, 2 ); // === 3
  1. Method

var my_object  =  {
	adder  :  add_one
};

my_object.adder( 1, 2 ); // === 3

Invoking a function

  1. Constructor
    Functions can be used to create new objects when invoked with the reserved word, new

var Person  =  function ( catchphrase ) {
	this.catchphrase  =  catchphrase;
};

Person.prototype.talk  =  function ( ) {
	alert( this.catchphrase );
}

// This is constructor invocation; new is *very* important
var matt  =  new Person( 'Kittens!' );

// And this is method invocation
matt.talk( );

Invoking a function

Why is new important?

Without new, Person is just a function that belongs to the global object, window:


// Don't do this!
var eryn  =  Person( 'Otters!' );
// → undefined

// eryn is now undefined (Person doesn't return anything)

// Even worse, window.catchprase is now 'Otters!':

alert( window.catchphrase );
// → 'Otters!'
  • Capitalize the names of constructor functions.
  • Invoke capitalized function names using new.

Invoking a function

  1. Apply
    Any function can be invoked as if it were invoked as a method on an object using apply:

var matt   =  new Person( 'Kittens!' );
var kevin  =  new Person( 'Puppies!' );

var deny_catchphrase  =  function ( ) {
	alert( 'I never say "' + this.catchphrase + '"' );
}

deny_catchphrase.apply( matt );
// → I never say "Kittens!"

deny_catchphrase.apply( kevin );
// → I never say "Puppies!"

matt.deny_catchphrase( );
// → Fails, no deny_catchphrase property on matt

deny_catchphrase( );
// → Fails, there is no this.catchphrase property on window

Invoking a function

Remember jQuery and .each( )?

jQuery uses apply under the hood to make this possible:


// All paragraphs should include unicorns anyway
$( 'p' ).each( function ( ) {
	$( this ).html( 'Unicorns!' );
} );
						

// Abbreviated from jQuery's source, core.js:
jQuery.each  =  function( obj, callback, args ) {
	var name,
		i = 0,
		length = obj.length;

	for ( ; i < length; ) {
		if ( callback.apply( obj[ i++ ], args ) === false ) {
			break;
		}
	}

	return obj;
}
